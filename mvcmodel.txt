MVC Model

<1> MVC 개념 및 의의

MVC ?
MVC는 M(Model) / V(View) / C(Controller) 로 구성함
디자인 패턴의 하나로 어떤 것을 개발하는 과정 속에 문제점들을 상황에 맞게 적용해서 쓸 수 있는 것을 정리하여 쉽게 사용 할 수 있도록 만든 것임
특히 협업을 하면서 유지보수를 하거나 역활분담을 할 때 가이드라인을 제시하는 방법으로 중요함

<2> MVC

1. Model : 애플리케이션의 정보 및 데이터 ex) 상수, 데이터베이스, 초기값

[무엇]을 할건인지를 정의
파일 저장소의 역활을 하며, DB와 상호작용
데이터 모델을 생성하고 이를 다루는 인터페이스 생성
데이터 내용을 검증하고 일반적인 오류는 컨트롤러에게 보고

2. View : 사용자 인터페이스 요소 ex) input 텍스트, 체크박스 항목
사용자가 직접 결과를 볼 수 있는 화면
데이터 및 객체의 입력, 출력
데이터베이스의 직접 접근을 피해야 함

3. Controller : 데이터(model)와 사용자인터페이스(view)를 연결짓는 매개체
사용자(user)-V-C-M-DB/서버
사용자가 데이터를 클릭하고, 모든 요청 에러를 처리
데이터를 화면에 표시하거나 데이터베이스와 로직을 직접 다루지 않도록 함
model과 view의 상활을 알고있어야 함


<3> MVC 장단점

1. 장점
사용자 인터페이스로부터 비즈니스 로직(프로그램의 논리구조)을 분리하여 어플리케이션의 시각적 요소와 그 이면에서 실행되는 비즈니스 로직을 수정 및 유지보수 가능
클라이언트의 요구사항에 대해 유연하게 대처가 가능(유연성)
각자의 역할에 집중할 수 있도록 업무 분할 효율성이 증대

2. 단점
기본 설계를 위해 클래스들이 많이 필요하여 그 과정 속에 속도가 느려질 수 있음
Model과 View가 완전히 독립적이지 않을 수 있음
